# 页面产品策略参考

每种页面类型都有特定的商业目标和内容组合模式。
设计页面前先读此文件，理解"该放什么"，再去模板看"长什么样"。

---

## 支付结果页

**商业目标**：结果页是运营黄金位——用户刚完成支付，注意力集中、心情放松，是推荐和教育的最佳时机。

**用户心智**：「我已经付完钱了，接下来呢？」

**标准内容组合**（从上到下）：

| 区域 | 内容 | 必要性 | 说明 |
|------|------|--------|------|
| 成功头部 | 成功图标 + 金额 + 商户 + 交易方式 | 必须 | 紧凑排列，不要拆成多张卡片 |
| 奖励/优惠区 | 支付有礼红包、优惠券、返现 | 强烈建议 | 结果页标配运营位，用激励驱动下一次行为 |
| 连接器 | 挂牌连接器（两个灰色竖条） | 可选 | 当奖励区和下方卡片有关联时使用 |
| 教育/推广卡片 | 新功能教育、服务推荐、交叉营销 | 强烈建议 | 用户注意力最集中的区域 |
| 底部按钮 | 完成/查看账单 | 必须 | 提供明确的退出路径 |

**教育卡片设计模式**：
- **Badge 标签**：蓝底白字胶囊（如「轻触支付」「服务标签」），标识卡片类型
- **高亮标题**：关键词用品牌蓝 `#1677ff` 突出（如「下次试试轻触支付，**快人一步**」）
- **插图区**：用 CSS 绘制、GE 动画或短视频展示操作过程
- **水平步骤流**：1→2→3 一行排列（比垂直列表更易扫读）
- **强 CTA**：口语化文案 + 全宽渐变按钮（如「查看方法」而不是「了解更多」）

**优惠券卡片设计模式**：
- 暖色渐变背景：`linear-gradient(135deg, #fff 40%, #FFF5ED 70%, #FFE8D6 100%)`
- 金额用红色大字高亮：`#FF4D4F`，fontSize 比正文大 1.5 倍
- CTA 用红色胶囊按钮（跟金额同色系）
- 副文案引导使用场景（如「下次试试轻触支付」「满 10 元可用」）

**人群变体策略**（可选，用于 A/B 测试）：

| 人群 | 标题 | 步骤 | CTA |
|------|------|------|-----|
| 默认（无明显问题） | 下次试试轻触支付，快人一步 | 解锁手机→靠近蓝环→支付完成 | 查看方法 |
| 安全敏感 | 解锁手机才能付，安全有保障 | 安全认证→加密支付→智能风控 | 了解多重保障 |
| NFC 未开启 | 打开NFC，下次就能轻触支付 | 开启NFC→解锁手机→轻触支付 | 去打开NFC |
| 靠近失败 | 技术全新升级，轻触支付就成功 | 解锁手机→手机NFC区域靠近设备 | 看看怎么用 |
| 频繁打开APP | 不用打开 App，解锁手机直接靠近设备 | 无需打开APP→解锁手机→轻触支付 | 试试看 |
| 电量敏感 | 手机性能优异，常开NFC不费电 | 日耗电<0.1%→后台零负担→常开更方便 | 去了解NFC电量真相 |

**融合模式**（券 + 教育合并为一张卡）：
- 顶部红包条：暖色渐变 + 红包 icon + 金额 + 「轻触支付时自动抵扣」
- 中间教育内容：同标准教育卡片
- CTA 按钮带角标：如「了解多重保障」+ 红色角标「领3元红包」

## 首页

**商业目标**：聚合多种功能入口和内容流，让用户快速找到所需功能。

**用户心智**：「我想用 XX 功能」或「看看有什么新的」

**标准内容组合**：

| 区域 | 内容 | 说明 |
|------|------|------|
| 搜索栏 | 搜索框 + 扫一扫 + 语音 | 页面最顶部，首要入口 |
| 功能宫格 | 4×2 或 5×2 图标网格 | 高频功能快捷入口（扫一扫、付款、出行等） |
| 运营卡片 | Banner + 活动卡片 | 品牌和运营推广位 |
| 服务推荐 | 纵向卡片流 | 个性化服务推荐 |

## 收付款页

**商业目标**：极速完成支付行为，减少一切干扰。

**用户心智**：「我要赶紧付钱/收钱」

**标准内容组合**：

| 区域 | 内容 | 说明 |
|------|------|------|
| 付款码 | 条形码 + 二维码 | 全屏品牌蓝背景，码区白底突出 |
| 切换入口 | 收款/付款切换 Tab | 底部或顶部 |
| 提示文案 | 安全提示 | 简短，不抢注意力 |

---

## 消息/通知页

**商业目标**：信息触达 + 引导回访。

**用户心智**：「看看有什么消息」

**标准内容组合**：

| 区域 | 内容 | 说明 |
|------|------|------|
| 分类 Tab | 服务通知/交易/活动 | 帮助用户快速定位 |
| 消息列表 | 头像 + 标题 + 摘要 + 时间 | 未读用红点或加粗 |
| 空状态 | 引导文案 + 操作入口 | 不只显示「暂无消息」 |

## 轻触支付页 / 确认付款页

**商业目标**：教育用户完成轻触支付动作，建立「轻触支付=方便」心智。确认页额外承担临门一脚转化。

**用户心智**：
- 支付页（等待态）：「我准备碰了」——需要清晰的视觉引导
- 确认页：「碰完了，确认付款？」——需要信任感和快速决策

**标准内容组合**：

| 区域 | 内容 | 必要性 | 说明 |
|------|------|--------|------|
| 渐变头部 | 浅蓝渐变 #E8F2FF→#D6E8FF，覆盖 65% 屏高 | 必须 | 安静氛围，不制造支付紧迫感 |
| 主标题 | 「轻触即付」28px Bold | 必须 | 教育型标题，字间距加宽 |
| 中心插图 | 手机+NFC波纹，占 42% 屏高 | 必须 | 视觉核心，手机倾斜15度暗示动作方向 |
| 底部白色区 | 圆弧过渡 border-radius 24px 24px 0 0 | 必须 | 支付页=Logo+安全文案；确认页=CTA |
| 导航右侧 | 「使用说明」文字链 | 可选 | 教育辅助入口 |

**双模式设计**：
- **waiting 模式**：Logo + 「安全支付中」，整页零决策点，纯等待状态
- **confirm 模式**：副标题 + 积分提示 + 蓝色渐变CTA，单一决策点

**关键约束**：
- 全页极简——教育型页面视觉传达优先于信息密度
- NFC 波纹用微紫色增加科技感（第三圈 `rgba(120,100,200,0.06)`）
- CTA 呼吸动效：轻微阴影脉动引导点击

## 轻触支付阵地页（Tab首页）

**商业目标**：轻触支付频道首页，承载品牌认知、用户教育、利益激励、场景引导四大职能。

**用户心智**：「看看轻触支付有什么用/有什么优惠」

**标准内容组合**：

| 区域 | 内容 | 必要性 | 说明 |
|------|------|--------|------|
| 渐变头部 | 浅蓝 #D6E8FF→#F5F5F5 + 主标题 26px + 设备副标题 | 必须 | 统一轻触支付蓝色调性 |
| 关系卡片 | 「好碰友」+ 陪伴天数 + 嵌入FAQ | 强烈建议 | 拟人化建立粘性 |
| 红包卡片 | 金额+过期时间+粉红CTA「去使用」 | 强烈建议 | 暖色CTA在蓝色基调中跳出 |
| 附近门店 | 门店数+品牌Logo+地图实景+浮层 | 强烈建议 | 真实距离具体化「附近优惠」 |
| 活动推荐 | 横滑活动卡片 | 可选 | 运营配置位 |
| 底部Tab栏 | 轻触支付/集卡换好礼/附近优惠 | 必须 | 子频道独立3Tab导航 |

**受阻教育变体**（当用户近期轻触支付失败时）：
- hero区替换为「受阻教育板块」
- 提示条 → 原因卡片（标签+标题+安抚+步骤+双CTA）→ 激励条
- 渐变从橙色改蓝色（冷静/信任），蓝色在受阻场景更合适
- 支持5种失败类型预设：NFC未开启/未解锁/距离太远/商户不支持/网络异常

**设计约束**：
- 所有区域标题带蓝色星标 ✦，暗示「智能推荐」
- 卡片流瀑布式，不用Tab切换（滑动浏览成本低于切换）
- 卡片间距统一 12px，圆角统一 20px

## 营销活动页（集卡/红包承接/附近优惠）

**商业目标**：促活+拉新——通过游戏化和利益驱动，让用户完成轻触支付。

**用户心智**：「有什么好处/怎么参与」

### 集卡活动页

| 区域 | 内容 | 说明 |
|------|------|------|
| 全屏渐变 | 紫蓝 #5B3FD9→#E8E0FF | 高饱和制造节日感 |
| 角色Tab | 5个角色胶囊选择 | 白底紫字选中态 |
| 3D卡片轮播 | 中心 220×260，两侧缩小 85% | 游戏化沉浸 |
| 金色CTA | 「去集卡」金色渐变 #FFD700→#FFA500 | 紫色背景上强对比 |
| 双任务卡片 | 做任务+刮刮乐并排 | 两条参与路径 |
| 红包横滑 | 浅红渐变卡片，金额24px | 红色=钱的心智 |

### 红包承接页

| 区域 | 内容 | 说明 |
|------|------|------|
| 数据背书 | 「多人已在使用」36px品牌蓝 | 社会认同先行 |
| 安全卖点 | 两行文案，关键词蓝色高亮 | 文案三层递进 |
| 优惠券轮播 | Swiper轮播白卡，金额48px超大 | 利益呈现核心 |
| 教学卡片 | 左图右文+CTA | 教育引导 |
| 信任条 | 保障品牌+安全文案 | 信任收尾 |

### 附近优惠商家页

| 区域 | 内容 | 说明 |
|------|------|------|
| 橙色渐变头部 | #FF8C38→#FFF5EC | 暖色=优惠/到店 |
| 大牌好券横滑 | 免费券优先展示 | 利益前置 |
| 分类Tab+快捷入口 | Tabs+胶囊标签行 | 筛选效率 |
| 商家卡片列表 | Logo+名称+距离+优惠+商品图+CTA | 高信息密度支撑决策 |
| 底部3Tab | 轻触支付/集卡/附近优惠 | 子频道导航 |

**通用营销约束**：
- 价格/金额统一用 `tabular-nums` + DINPro 字体族
- 激励色（红包红 `#FF3141`、优惠橙 `#FF4D00`）只用于金额和CTA
- 品牌蓝 `#1677FF` 仅出现在导航/选中态，不与激励色混用

## 安全感教育页

**商业目标**：消除用户对轻触支付安全性的疑虑，逐层建立信任，降低支付转化障碍。

**用户心智**：「轻触支付安全吗？手机丢了怎么办？」

**标准内容组合**：

| 区域 | 内容 | 必要性 | 说明 |
|------|------|--------|------|
| 数据背书 | 「多人已在使用」+ 主标题 | 必须 | 社会认同建立基线 |
| 分段Tab | 4个安全主题锚点 | 必须 | 长页面导航 |
| FAQ问答卡片 | Q/A标签+场景插图+详情链接 | 必须 | 用户疑问驱动 > 陈述 |
| 安全特性 | 3列等分（认证/加密/防控） | 强烈建议 | 一屏可见3个关键点 |
| 保障承诺 | 保障品牌+赔付说明 | 强烈建议 | 保险品牌比自说自话更可信 |
| 媒体报道 | 横滑媒体卡片（媒体/安全机构） | 可选 | 第三方权威=信任链终环 |

**关键约束**：
- 长页面~2400px（约3屏滚动），不用弹窗——安全教育需充分展开
- 信任递进：数据认同 → 问题解答 → 技术保障 → 品牌背书 → 媒体佐证
- Q标签蓝底 `#1677FF`、A标签绿底 `#00B578`，视觉区分问答
- 浅蓝色调与轻触支付页同色系：蓝色=安全/信任心理暗示

## 通用原则

1. **结果页 = 运营黄金位**：永远要有优惠/推荐内容，不要只放成功信息
2. **教育 ≠ 说明书**：用场景化语言（「查看方法」）而非功能描述（「了解NFC支付功能」）
3. **激励先行**：先给利益（3元红包），再说教育（怎么用）
4. **步骤要少**：3步以内，水平排列
5. **CTA 口语化**：「查看方法」「去领取」「试试看」，不要「了解更多」「查看详情」
6. **变体思维**：考虑不同用户群体的痛点，准备 2-3 个变体方案
