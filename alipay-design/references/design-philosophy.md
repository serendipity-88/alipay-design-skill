# 支付宝设计理念与视觉规范

## 设计哲学："轻Design"

三个核心关键词：**轻灵**、**自然**、**年轻**。让界面像呼吸一样自然，减少视觉噪音，内容优先呈现。

---

## 五大页面设计模式

以下模式面向支付、金融、生活服务类页面整理，每种模式包含设计意图和具体参数。

### 模式 1：情绪头部型（Emotion Header）

**典型页面**：支付结果页、收付款页
**设计意图**：用大面积品牌色渐变传递情绪——成功的安心感、支付的信任感。蓝色覆盖 40-100% 屏幕高度，白色文字浮于其上，形成强烈的品牌沉浸。

**参考参数**：

| 属性 | 支付结果页 | 收款码页 |
|------|-----------|---------|
| 渐变覆盖 | 49.4% 屏高（750×802） | 100% 屏高（750×1624） |
| 头部渐变 | `#429fff → #1677ff → #f5f5f5` | `#1677ff → #1677ff → #30a2ff` |
| 奖励区渐变 | `#569cff → #edf4ff` | — |
| 阴影 | `rgba(88,123,140,0.08)` y=20 blur=40 | 无阴影 |
| 卡片圆角 | 20-32px | 16-24px |
| 文字色 | 白色 ×14，品牌蓝 ×4 | 白色 ×10 |

**CSS 骨架**：
```css
/* 情绪头部 — 成功结果页 */
.emotion-header {
  background: linear-gradient(to bottom, #429fff 0%, #1677ff 74%, #f5f5f5 100%);
  min-height: 49vh;
}
/* 情绪头部 — 全屏沉浸（收款码） */
.emotion-header--full {
  background: linear-gradient(135deg, #1677ff 0%, #30a2ff 100%);
  min-height: 100vh;
}
.emotion-header .result-card {
  background: #fff;
  border-radius: 24px;
  box-shadow: 0 20px 40px rgba(88, 123, 140, 0.08);
  margin: 0 24px;
}
```

**何时使用**：操作完成反馈、需要强烈品牌感知的场景、单一聚焦任务（如收付款）。

### 模式 2：信息密集型（Information Dense）

**典型页面**：首页、理财、出行
**设计意图**：头部用品牌渐变（15-28% 屏高）建立认知锚点，随即过渡到密集的内容流。通过卡片分组 + 网格排列管理信息密度，靠阴影和圆角区分层级而非线条。

**参考参数**：

| 属性 | 首页 | 理财 | 出行 |
|------|------|------|------|
| 图层数 | 998 | 1998 | 570 |
| 文本层 | 93 | 200 | 44 |
| 渐变数 | 121 | 37 | 109 |
| 头部渐变 | `#005eff → #1677ff → #0097ff` | `#009eff → #1677ff → #005eff` | `#fff → #e5f3ff → #e4f3ff` |
| 头部高度 | 412px（25.4%） | 458px（28.2%） | 全屏淡蓝底 |
| Body 渐变 | `#ffffff → #f0f3f7` | — | — |
| 卡片数 | 35 | 38 | 36 |
| 主卡片圆角 | 24px | 24-28px | 24-34px |
| 阴影 | `rgba(0,95,221,1)` y=2 blur=5 | `rgba(0,0,0,0.04)` y=0 blur=16 | `rgba(254,255,255,0.4)` y=1 blur=3 |
| 主文字色 | #333 ×35, #666 ×11 | #fff ×117, #555 ×20 | #333 ×21, #1677ff ×7 |

**CSS 骨架**：
```css

.dense-header {
  background: linear-gradient(180deg, #005eff 0%, #1677ff 50%, #0097ff 100%);
  height: 206px; /* 约 25% 屏高，基于 375pt */

.dense-body {
  background: linear-gradient(to bottom, #ffffff, #f0f3f7);
  padding: 0 12px;

.content-card {
  background: #fff;
  border-radius: 24px;
  box-shadow: 0 2px 5px rgba(0, 95, 221, 0.08);
  padding: 16px;
  margin-bottom: 12px;
}
/* 宫格入口（首页九宫格） */
.grid-entry {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  gap: 0;
  text-align: center;
}
```

**何时使用**：需要在一屏内展示多种功能入口和内容流的聚合页。

### 模式 3：轻量列表型（Light List）

**典型页面**：消息、我的、卡包
**设计意图**：淡化头部色彩（使用浅色渐变而非品牌蓝），让注意力集中在列表内容本身。用低饱和度的头部营造安静、有序的阅读氛围。

**参考参数**：

| 属性 | 消息 | 我的 | 卡包 |
|------|------|------|------|
| 图层数 | 765 | 432 | 337 |
| 头部渐变 | `#edf2f8 → #c8ddfc` | `#f0f3f7 → #c8ddfc` | 纯白 |
| 阴影 | `rgba(0,35,85,0.11)` y=5 blur=14 | 无阴影 | `rgba(215,222,233,0.74)` y=-7 blur=31 |
| 卡片圆角 | 24px | 24px | 24-40px |
| 主文字色 | #333 ×17, #999 ×17 | #333 ×28 | #999 ×17, #f93a4a ×15 |
| 特色 | 头像渐变 `#09bfd0 → #6ea2fd` | 无阴影，纯间距分区 | 优惠券用 `#f93a4a` 强调 |

**CSS 骨架**：
```css

  background: linear-gradient(to bottom, #edf2f8, #c8ddfc);

  box-shadow: none;
  margin-bottom: 8px;
}
```

**何时使用**：内容为主的浏览页、设置/个人中心、消息/通知列表。

### 模式 4：沉浸暗色型（Immersive Dark）

**典型页面**：视频沉浸式
**设计意图**：暗色背景让视觉内容（视频、图片）成为绝对主角。大量使用阴影（73 个）营造层次，白色文字统一色调，减少一切与内容竞争注意力的元素。

| 卡片圆角 | 200px（全圆角胶囊按钮） |
| 文字色 | 白色 ×23（全部白色） |

**CSS 骨架**：
```css

```

### 模式 5：极简表单型（Minimal Form）

**典型页面**：注册登录、搜索
**设计意图**：极度克制——几乎无渐变、无阴影、大量留白。让用户聚焦于唯一任务（输入）。视觉元素越少，认知负担越低，转化率越高。

**参考参数**：

| 属性 | 注册登录 | 搜索前置页 |
|------|---------|-----------|
| 图层数 | 71-73 | 248 |
| 渐变 | 仅 1 个（按钮） | 3 个 |
| 阴影 | 0 | 0 |
| 卡片数 | 3-4 | 13 |
| 文字色 | #999 ×5, #4b6b99 ×3 | — |
| 搜索头部 | — | `#fff → #1677ff`（23.2% 屏高） |

**CSS 骨架**：
```css
/* 极简表单型 */
.minimal-form {
  background: #fff;
  padding: 32px 24px;
}
.minimal-form .title {
  font-size: 24px;
  color: #333;
  font-weight: 600;
  margin-bottom: 32px;
}
.minimal-form .input-field {
  border: none;
  border-bottom: 1px solid #eee;
  padding: 16px 0;
  font-size: 17px;
  width: 100%;
}
.minimal-form .hint-text {
  color: #999;
  font-size: 13px;
  margin-top: 12px;
}
.minimal-form .primary-btn {
  background: linear-gradient(135deg, #1677ff, #179eff);
  border-radius: 24px;
  color: #fff;
  width: 100%;
  height: 48px;
}
/* 底边框输入框 + suffix 按钮（如"获取验证码"）
 * ⚠️ 不要用 position: absolute 放 suffix，会与 hint/error 定位冲突
 * 正确做法：flex 行包裹 input + suffix，底边框和状态色挂在容器上 */
.minimal-form .input-row {
  display: flex;
  align-items: center;
  border-bottom: 1px solid #eee;
  transition: border-color 300ms ease;
}
.minimal-form .input-row:focus-within { border-bottom-color: #1677ff; }
.minimal-form .input-row.error { border-bottom-color: #ff3141; }
.minimal-form .input-row.success { border-bottom-color: #00b578; }
.minimal-form .input-row .input-field {
  flex: 1;
  border: none;
  outline: none;
  /* 覆盖上方 .input-field 的 border-bottom，由 .input-row 控制 */
}
.minimal-form .input-row .suffix {
  flex-shrink: 0;
  padding-bottom: 12px;
  font-size: 14px;
  color: #1677ff;
  font-weight: 500;

```

**何时使用**：需要用户输入的任务页、登录注册、搜索。

## 扩展页面模式

以下模式是五大基础模式的细分变体，在页面策略和代码模板中使用。它们继承基础模式的视觉参数（圆角、阴影、间距），仅在情绪策略和内容组合上有差异。

| 扩展模式 | 基础模式 | 差异点 | 典型页面 |
|----------|----------|--------|----------|
| 沉浸教育型 | 情绪头部型 | 浅蓝渐变（非深蓝），大面积插图占 50%+ 屏高，文案以教育引导为主 | 轻触支付页、确认付款页 |
| 营销承接型 | 情绪头部型 | 暖色渐变头部，数据背书区（大数字），优惠券卡片密集，CTA 强引导 | 红包承接页 |
| 活动运营型 | 信息密集型 | 彩色渐变背景（紫蓝/橙红），游戏化元素（卡片轮播/刮刮乐），底部 Tab 栏 | 集卡活动、附近优惠商家 |
| 情绪安抚型 | 轻量列表型 | 受阻场景专用，暖色激励文案替代红色警告，紧凑信息 + 单 CTA | 受阻教育页 |
| 长页面教育型 | 轻量列表型 | 蓝色渐变头部 + 多段折叠教育内容，FAQ 区块，底部保障承诺 | 安全感教育页 |

使用方式：选择扩展模式时，先继承其基础模式的圆角/阴影/间距参数，再叠加差异点的情绪策略。

## 设计意图层：四个维度

### 1. 色彩情绪策略

| 策略 | 色值 | 何时用 |
|------|------|--------|
| 信任蓝 | `#1677ff` 品牌主色 | 支付、结果、主按钮 |
| 深蓝权威 | `#005eff → #1677ff` | 首页/理财头部，传递专业可靠 |
| 浅蓝安静 | `#edf2f8 → #c8ddfc` | 消息/我的头部，不抢注意力 |
| 功能橙红 | `#f93a4a`, `#ff6430` | 优惠券、限时标签，制造紧迫感 |
| 中性灰阶 | `#333 → #666 → #999` | 信息层级：标题 → 正文 → 辅助 |
| 暗色沉浸 | `#000 → #111` | 视频/内容消费，让内容做主角 |

### 2. 视觉重量分布

页面从上到下的视觉重量遵循 **"重头轻身"** 或 **"均匀流动"** 两种节奏：

- **重头轻身**（结果页、收款码）：头部渐变 ≥40% 屏高，传递强情绪后快速过渡到轻量内容区
- **均匀流动**（首页、理财）：头部 25-28%，内容区通过卡片+网格均匀分布视觉重量
- **无头极简**（登录、搜索）：几乎无视觉重量，全部让给内容

### 3. 空间叙事

每个页面是一个叙事流：**情绪建立 → 信息传递 → 行动引导**。

- **第一幕 · 品牌头部**（0-25%）：渐变色建立情绪基调
- **第二幕 · 核心内容**（25-75%）：卡片组织信息，白底减少干扰
- **第三幕 · 操作收尾**（75-100%）：按钮吸底，明确下一步

支付结果页将第一幕扩展到 49%，因为"成功"本身就是最重要的信息。

### 4. 材质层级

| 层级 | 材质 | 阴影 | 圆角 | 用途 |
|------|------|------|------|------|
| L0 背景 | `#f5f5f5` 或渐变 | — | — | 页面底色 |
| L1a 卡片（密集型） | 白色 `#fff` | `0 2px 12px rgba(0,0,0,0.06)` | 12-16px | 信息密集型列表卡片 |
| L1b 卡片（情绪型） | 白色 `#fff` | `0 4px 20px rgba(90,110,149,0.08)` | 20-24px | 情绪头部型/营销承接型主卡片 |
| L2 浮层 | 白色 + 毛玻璃 | `0 8px 32px rgba(90,110,149,0.15)` | 16-32px | 弹窗/底部面板/评论面板 |
| L3 强调 | 品牌渐变 | 品牌色阴影 `rgba(22,119,255,0.25)` | 胶囊 | CTA 按钮 |

## 通用视觉规则

### 字体规范（移动端 750px 宽设计稿）

**字体族**：`PingFangSC`（主要）、`DINPro` / `tabular-nums`（金额数字）

**字号体系**（设计稿 @2x 值 → 实际 pt 值）：

| 语义 | 设计稿值 | 实际 pt | 字重 | 使用场景 |
|------|-----------|---------|------|----------|
| 超大标题 | 60px | 30pt | Bold | 登录页主标题（极少用） |
| 大数字 | 52px | 26pt | DINPro-Medium | 理财金额展示 |
| 页面标题 | 40px | 20pt | Medium | 价格标签、登录输入提示 |
| 卡片标题 | 36px | 18pt | Medium | 导航栏标题、区块标题 |
| 正文大 | 34px | 17pt | Regular | 收款码说明文字 |
| 列表标题 | 32px | 16pt | Regular | 消息列表主标题 |
| **正文标准** | **30px** | **15pt** | Regular | 我的页列表、说明文字 |
| **正文常用** | **28px** | **14pt** | Medium/Regular | 卡片标题、按钮文字（最高频之一） |
| 次要内容 | 26px | 13pt | Regular | 列表描述、标签文字 |
| **基础文字** | **24px** | **12pt** | Regular | 辅助说明、时间戳（**全局最高频**） |
| 辅助标签 | 22px | 11pt | Regular | 视频字幕、小标签 |
| 极小文字 | 20px | 10pt | Regular | 底部提示、版权信息 |

**字重使用规律**：
- `PingFangSC-Regular`：正文、描述、辅助信息（占 70%+）
- `PingFangSC-Medium`：卡片标题、按钮、强调文字
- `PingFangSC-Semibold`：导航栏标题、数字金额
- `DINPro-Medium` / `tabular-nums`：金额数字（理财、出行）

**文字色层级**（从最常用到最少用）：
- `#333333`（主文本）→ `#666666`（次文本）→ `#999999`（辅助/占位符）→ `#cccccc`（禁用）
- 白底上：`#333` 为主，深蓝/渐变背景上：`#ffffff` 为主
- 功能色文字：`#1677ff`（链接）、`#f93a4a`（价格/错误）、`#1d3568`（深蓝辅助）

### 阴影规范

| 场景 | box-shadow |
|------|-----------|
| 标准卡片 | `0 4px 12px rgba(90, 110, 149, 0.12)` |
| 轻量卡片（理财） | `0 0 16px rgba(0, 0, 0, 0.04)` |
| 消息列表卡片 | `0 5px 14px rgba(0, 35, 85, 0.11)` |
| 结果页卡片 | `0 20px 40px rgba(88, 123, 140, 0.08)` |
| 沉浸式浮层 | `0 0 128px rgba(0, 0, 0, 0.06)` |
| 卡包优惠券 | `0 -7px 31px rgba(215, 222, 233, 0.74)` |

**规律**：阴影颜色使用冷灰蓝调（不是纯黑），透明度 4-12%，模糊值 12-40px。

### 间距系统（4px 基准）

| 级别 | 值 | 用途 |
|------|-----|------|
| xs | 4px | 图标与文字间距 |
| sm | 8px | 元素内间距 |
| md | 12px | 卡片内间距、卡片间距 |
| lg | 16px | 区块间距 |
| xl | 24px | 大区块分隔、页面水平边距 |
| xxl | 32px | 页面顶部/底部留白 |

### 圆角规范（按页面模式分级）

**核心原则**：元素越小圆角越小，元素越大圆角越大，形成视觉递进。

| 元素 | 情绪头部型/营销承接型 | 信息密集型/轻量列表型 | 极简表单型 |
|------|----------------------|---------------------|-----------|
| 主内容卡片 | 20-24px | 12-16px | — |
| 小功能卡片/插图区 | 12-16px | 8-12px | 8px |
| 按钮（标准） | 8px | 4-8px | 4px |
| 按钮（胶囊） | 200px | 200px | 200px |
| 标签/Tag | 4-8px | 4px | 4px |
| 弹窗/浮层 | 32px | 16px | 12px |
| 底部面板 | 16px 16px 0 0 | 16px 16px 0 0 | — |
| 优惠券卡片 | 24-32px | 12-16px | — |
| 头像 | 100px（全圆） | 100px（全圆） | 100px（全圆） |

### 动效 Token

| Token | 用途 | 值 |
|-------|------|-----|
| `duration-fast` | 微交互 | `150ms` |
| `duration-normal` | 标准过渡 | `300ms` |
| `duration-slow` | 页面切换 | `450ms` |
| `easing-standard` | 通用 | `cubic-bezier(0.4, 0, 0.2, 1)` |
| `easing-decelerate` | 进入 | `cubic-bezier(0, 0, 0.2, 1)` |
| `easing-accelerate` | 退出 | `cubic-bezier(0.4, 0, 1, 1)` |

### 视觉降噪原则

- 减少分割线，用间距和卡片替代
- 卡片用阴影区分层级，不用描边
- 图标用线性风格（不用填充），保持轻盈
- 同屏动效不超过 2 个同时进行
- AI 场景用蓝紫渐变：`linear-gradient(135deg, #1677FF, #7B61FF)`

## 六大设计原则速查

| 原则 | 落地要点 |
|------|----------|
| 自然真实 · 有层次 | 用阴影和模糊表达层级，不用描边 |
| 轻巧灵动 · 智能感 | 适度动效反馈，AI 场景用蓝紫渐变 |
| 轻盈有序 · 更统一 | 遵循 4px 间距系统，组件规范统一 |
